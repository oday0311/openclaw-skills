# Whisper模型大小对比

## 模型规格

| 模型 | 磁盘大小 | 内存占用 | 准确度 | 速度 | 推荐场景 |
|------|----------|----------|--------|------|----------|
| **tiny** | 39 MB | ~100 MB | 低 | 最快 | 快速测试，实时转录 |
| **base** | 74 MB | ~200 MB | 中等 | 快 | **本工作流默认**，日常使用 |
| **small** | 244 MB | ~500 MB | 良好 | 中等 | 需要较高准确度 |
| **medium** | 769 MB | ~1.5 GB | 高 | 较慢 | 专业转录，重要内容 |
| **large** | 1550 MB | ~3 GB | 最高 | 最慢 | 研究用途，最高质量 |

## 为什么选择base模型？

本工作流固定使用**base模型**，原因如下：

### 1. 磁盘空间友好
- base模型仅74MB，是medium模型的1/10
- 适合小磁盘环境（如VPS、云服务器）
- 下载速度快，部署方便

### 2. 性能平衡
- 准确度：满足大多数日常抖音视频转录需求
- 速度：处理1分钟音频约需10-20秒
- 内存：仅需约200MB内存，适合低配置环境

### 3. 实际效果
对于抖音视频的典型场景：
- 视频长度：通常15秒到3分钟
- 音频质量：中等（可能有背景音乐）
- 语言：主要是中文普通话
- 背景噪音：可能有环境噪音

base模型在这些条件下表现良好，准确率可达85-90%。

## 如何更改模型

如果你有足够的磁盘空间和内存，可以修改模型：

### 方法1：修改主脚本
编辑 `scripts/douyin_asr_workflow.py`：
```python
whisper_cmd = [
    "whisper",
    audio_path,
    "--model", "base",  # 改为 "small", "medium", "large"
    # ...
]
```

### 方法2：使用环境变量（高级）
可以修改脚本支持环境变量：
```python
import os
model_size = os.getenv("WHISPER_MODEL", "base")  # 默认base
```

然后通过环境变量指定：
```bash
export WHISPER_MODEL="small"
python3 scripts/douyin_asr_workflow.py "https://v.douyin.com/xxxxxxx/"
```

## 磁盘空间管理

### 模型缓存位置
Whisper模型默认下载到：
```
~/.cache/whisper/
```

### 清理模型缓存
如果需要释放磁盘空间：
```bash
# 查看模型缓存
ls -lh ~/.cache/whisper/

# 删除特定模型
rm ~/.cache/whisper/base.pt

# 删除所有模型缓存
rm -rf ~/.cache/whisper/
```

### 更改缓存位置
可以通过环境变量更改缓存位置：
```bash
export XDG_CACHE_HOME="/path/to/your/cache"
# 或
export WHISPER_CACHE_DIR="/path/to/whisper/cache"
```

## 性能优化建议

### 对于小磁盘环境
1. **坚持使用base模型** - 最佳平衡
2. **定期清理缓存** - 删除不需要的模型
3. **使用外部存储** - 如果有外部硬盘

### 对于性能要求高的场景
1. **使用SSD** - 加快模型加载速度
2. **增加内存** - 减少磁盘交换
3. **批量处理** - 避免重复加载模型

## 常见问题

### Q: base模型准确度够用吗？
A: 对于大多数抖音视频（日常对话、教程、娱乐内容），base模型准确度足够。如果处理专业内容（技术讲座、重要会议），可考虑small或medium模型。

### Q: 可以自动选择模型吗？
A: 当前版本固定使用base模型以简化配置。未来版本可考虑根据视频长度自动选择。

### Q: 模型下载失败怎么办？
A: 检查网络连接，或手动下载模型放到缓存目录。

### Q: 如何知道当前使用的模型？
A: 查看工作流日志，会显示使用的模型参数。
